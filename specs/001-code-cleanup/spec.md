## User Scenarios & Testing *(mandatory)*

### User Story 1 - Удаление мёртвого кода (Priority: P1)

Как разработчик, я хочу убрать неиспользуемый код из `page1.html`, чтобы файл стал короче, проще для чтения и поддержки.

**Why this priority**: Мёртвый код путает при чтении, создаёт ложное впечатление о функциональности и увеличивает размер файла, загружаемого пользователем.

**Independent Test**: Открыть конфигуратор в браузере и пройти весь пользовательский путь: выбрать конфигурацию (2/4 подушки, 1/2 одеяла), нажать на элемент, выбрать размер, выбрать цвет, посмотреть состав заказа, добавить в корзину. Всё MUST работать идентично до и после изменений.

**Acceptance Scenarios**:

1. **Given** код содержит неиспользуемые функции и переменные, **When** они удалены, **Then** все пользовательские сценарии конфигуратора работают без изменений
2. **Given** файл содержит закомментированный код, **When** комментарии-заглушки удалены, **Then** файл становится короче и чище без потери функциональности

---

### User Story 2 - Удаление избыточных debug-логов (Priority: P2)

Как разработчик, я хочу убрать чрезмерное количество `console.log` вызовов, которые предназначались для отладки и не нужны в продакшене.

**Why this priority**: Многочисленные логи засоряют консоль, потенциально замедляют выполнение и раскрывают внутреннюю логику приложения конечным пользователям.

**Independent Test**: Открыть конфигуратор, выполнить полный пользовательский путь, убедиться что в консоли нет избыточного потока сообщений, при этом критичные ошибки по-прежнему логируются.

**Acceptance Scenarios**:

1. **Given** код содержит десятки `console.log` вызовов, **When** отладочные логи удалены, **Then** консоль чиста при нормальной работе, а `console.error` для реальных ошибок сохранён
2. **Given** дублированный `console.log('Start Script 3')`, **When** дубль удалён, **Then** сообщение выводится ровно один раз

---

### User Story 3 - Исправление дублирующихся ID в данных товаров (Priority: P3)

Как владелец магазина, я хочу чтобы у каждого товара был уникальный идентификатор, чтобы избежать потенциальных ошибок при работе с корзиной и аналитикой.

**Why this priority**: Дублирующиеся ID (`"id": "pillows14"` используется у 6 разных товаров в разделе prints) могут привести к неправильной работе при переходе на динамическую загрузку товаров с сервера.

**Independent Test**: Проверить что каждый объект товара в `products` имеет уникальный `id`. Убедиться что выбор любого из ранее дублирующихся товаров корректно добавляет в корзину именно выбранный товар.

**Acceptance Scenarios**:

1. **Given** несколько товаров prints-наволочек имеют одинаковый `id: "pillows14"`, **When** ID исправлены на уникальные, **Then** каждый товар идентифицируется однозначно

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Система MUST удалить все функции, которые объявлены но нигде не вызываются
- **FR-002**: Система MUST удалить все переменные, которые объявлены но нигде не читаются
- **FR-003**: Система MUST удалить весь закомментированный код
- **FR-004**: Система MUST сохранить все `console.error` и `console.warn` вызовы для реальных ошибок
- **FR-005**: Система MUST удалить отладочные `console.log` вызовы
- **FR-006**: Система MUST присвоить уникальные ID всем товарам с дублирующимися ID
- **FR-007**: Система MUST сохранить полную работоспособность конфигуратора после всех изменений

### Конкретные элементы для удаления/исправления

Обнаруженные проблемы:

1. **Закомментированный объект `sizes`** — неиспользуемый массив размеров
2. **Дублированный `console.log('Start Script 3')`** — одинаковый вызов в двух местах
3. **Функция `getSizesArray()`** — нигде не вызывается; логика дублирует `showSizeOptions`
4. **Функция `showPalette()`** — обёртка над `loadSwatches()`, нигде не вызывается
5. **Отсутствует HTML-элемент `#notification`** — необходимо добавить `<div id="notification" class="notification"></div>` в HTML, чтобы функция `showNotification` заработала
6. **Переменная `currentCategory`** — установлена в `'solids'`, нигде не читается
7. **Функция `updateAllImagesForConfig()`** — объявлена, но не вызывается; используется `updateImagesForConfig`
8. **Функции `loadFromLocalStorage()` и `saveToLocalStorage()`** — `load` не вызывается, `save` вызывается но данные никогда не читаются; удалить обе и все вызовы `saveToLocalStorage`
9. **Ссылки на `window.applyToAllPillows` и `window.applyToAllBlankets`** — проверяются в коде, но нигде не устанавливаются (всегда `undefined`)
10. **Дублирующиеся ID `"pillows14"`** — у 5 товаров: ЛЕЙКА, ЛУГ, РАСКРАСКА, ТИХИЙ ЧАС, ТОЧКА ЗАПЯТАЯ
11. **Дублирующийся ID `"mattresses9"`** — у товаров ТОПЛЕНОЕ МОЛОКО и НЕБО в mattresses.solids
12. **Отсутствует HTML-элемент `<div id="notification">`** — MUST добавить элемент для работы уведомлений

### Key Entities

- **products** — объект с данными всех товаров (pillows, blankets, mattresses), каждый с уникальным id, name, image[], sku, sizes[], picture
- **currentSelections** — текущие выборы пользователя по каждому слоту
- **userSelections** — явно подтверждённые пользователем выборы для оформления заказа

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Количество строк кода уменьшается минимум на 50 строк
- **SC-002**: Все ID товаров в объекте `products` уникальны
- **SC-003**: Полный пользовательский путь конфигуратора (выбор конфигурации → выбор элемента → размер → цвет → заказ → корзина) работает без ошибок
- **SC-004**: В консоли браузера при нормальной работе нет избыточных отладочных сообщений

## Assumptions

- Функции, не вызываемые из нашего кода в `page1.html`, также не вызываются из внешнего Tilda-контекста (за исключением `window.proceedToCheckout`, `window.showNotification` и других явно глобальных функций через `window.`)
- `loadFromLocalStorage` и `saveToLocalStorage` не вызываются из других скриптов на странице Tilda; обе безопасно удалить
- Уведомления (`showNotification`) не работают из-за отсутствия HTML-элемента `#notification`; решение — добавить элемент, функцию сохранить
- Условия `applyToAllPillows` / `applyToAllBlankets` — это запланированная, но нереализованная функциональность, которую безопасно удалить
